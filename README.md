## 📊 프로젝트 개요 (Overview)

작전 지원 환경은 변동성이 높아 신속하고 유연한 예외 상황 대응이 필수적입니다. 하지만 매일 타워에서 하달되는 한글(HWP) XX반 작업조원용 스케줄(Excel)로 변환하는 데 **매일 1시간 이상의 단순 반복 수작업**이 소요되고 있었습니다.

이를 해결하기 위해, 원본 데이터를 입력하면 원클릭으로 데이터 정제부터 매핑, 서식 지정까지 완료되는 **VBA 기반 자동화 프레임워크를 직접 설계 및 구축**하였습니다.

### 🔄 AS-IS vs TO-BE (프로세스 혁신)

| 구분 | 프로세스 단계 | 소요 시간 | 리스크 및 문제점 |
| :--- | :--- | :--- | :--- |
| **AS-IS**<br>(수작업) | HWP 복사 → 불필요 열 수동 삭제 → Xx-xx 및 조 수동 매핑 → 출력 서식 설정 | **일 1시간 이상** | 단순 반복 작업으로 인한 휴먼 에러 발생 가능성.<br>긴급 스케줄 변동 시 병목 현상 발생. |
| **TO-BE**<br>(자동화) | HWP 복사 → **[매크로 실행]** → 특이사항(예외) 최종 검수 | **1분 이내** | 데이터 변환 과정의 결함률(찐빠) **0% 달성**.<br>확보된 시간을 '예외 상황 통제'에 집중. |

## 💻 핵심 자동화 로직 및 기술적 주안점 (Key Features)

본 매크로는 단순 녹화(Record) 방식이 아닌, 기업 환경의 RPA(Robotic Process Automation) 프레임워크 설계 방법론을 적용하여 하드코딩되었습니다.

1. **동적 문자열 파싱 및 데이터 정제 (String Parsing)**
   * 불필요한 '0'이 포함된 연속된 문자열(예: `63077063`)에서 의미 있는 고유 번호 2자리만 정확히 추출.
   * `Mid` 함수와 `Step 3` 반복문을 활용하여 데이터를 정제하고, `vbLf`를 삽입해 셀 내 줄바꿈 처리를 자동화.
2. **오류 방지를 위한 2단계 매핑 알고리즘 (VLOOKUP Fail-safe)**
   * 참조 데이터(텍스트 vs 숫자) 간의 타입 불일치로 인한 N/A 에러 방지.
   * 1차 텍스트 매칭 실패 시, `IsNumeric` 판별 및 `Val()` 함수를 통해 숫자형으로 강제 변환하여 2차 매칭을 시도하는 안전망 로직 구현.
3. **비정형 텍스트 동적 추출 (Unstructured Data Extraction)**
   * `InStr` 함수를 사용하여 텍스트 내 괄호 `(...)`의 시작과 끝 인덱스를 동적으로 추적.
   * 조건문을 통해 괄호 쌍이 온전히 존재할 때만 내부의 T/O 데이터를 추출하여 병합.
4. **사용자 경험(UX) 및 변화 관리 (UX & Change Management)**
   * IT 비전문가인 군 간부 및 후임병들의 불신을 해소하기 위해 `Application.StatusBar`와 `DelayTime`을 활용, 매크로 진행 상황(1~7단계)을 실시간으로 시각화.
   * 참조 시트 누락 등 예외 상황 발생 시, 프로세스가 강제 종료되지 않고 명확한 안내 메시지(`MsgBox`)를 띄우도록 에러 핸들링(`On Error Resume Next`) 적용.
